I've been thinking of how exactly I should implement blogs. Right now, I haven't implemented any at all BUT I have formatted them in a [`gray-matter`](https://www.npmjs.com/package/gray-matter) compatible way. And though I hardly know what `gray-matter` is capable of, I was hypothesising that what I am about to suggest might actually work.

I kinda want to catagorise blog posts, not only by tags, but by a _main_ tag ✨✨. So if its a blog post that documents the progress and thoughts while working on a project, I'd want it to have the main tag of the project name, in this case `portfolio-website`. But I don't want this to be very strict, so the value of `main-tag` can be undefined. 

The main reason why i'd want this feature is because I eventually want to have this `main-tag` meta data in the website header while not impeding on the actual url. (so the blog post is like a standalone thing)
